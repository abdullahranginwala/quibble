# Decisions log (append-only)

- 2026-07-07 M0: `go.mod` directive is `1.25`, not `1.24` — `go-internal@v1.15` (testscript) requires 1.25; toolchain installed is 1.26.4. Conventions file said 1.24; bump was forced by the allowed dep set.
- 2026-07-07 M0: `config.Default()` authors are `human`/`agent` (generic); `quibble init` keeps them — users personalize in config. Validate() additionally requires human ≠ agent since the resolve policy gate depends on distinguishing them.
- 2026-07-07 M0: glob matcher also skips all dot-directories (not just `.quibble`) during doc walks — hidden dirs as doc sources are out of scope.
- 2026-07-07 M1: `NormalizeBlocks` signature is `func(src []byte) (text string, blocks []Block)` — the spec exports the name but leaves the signature open; this shape is the single source of truth for both `Doc.Text` and `Doc.Blocks`, so `RenderDoc` and `pkg/comment` consume the identical function. Frozen by golden `testdata/golden/normalize.json`.
- 2026-07-07 M1: paper theme assets live at `pkg/render/themes/paper/` (embedded via `go:embed themes`), not `web/themes/`. `go:embed` patterns cannot cross package boundaries (no `..`), and the M1 hard constraint forbids touching files outside `pkg/render`. Assets are trivially relocatable to a `web/` package later if a shared frontend tree is wanted.
- 2026-07-07 M1: block enumeration — a list contributes one block per *direct* list item; nested sub-lists fold their text into the containing item's block (not enumerated separately). Blocks with empty normalized text (e.g. thematic breaks, image-only paragraphs with empty alt) are excluded from `Doc.Text`/`Doc.Blocks` and receive no `data-qbl`. Spec was silent on nesting and empties; pinned by the kitchen-sink goldens.
- 2026-07-07 M1: chroma styles are `github` (light) / `github-dark` (dark), emitted as classes and scoped under `[data-qbl-scheme="dark"]` plus a `prefers-color-scheme: dark` media query. Custom `data-qbl` attributes ride goldmark's default block renderers (goldmark renders any `data-*` attribute regardless of its per-node filter), so only heading + code-block rendering is overridden.
- 2026-07-07 M1: renderer uses `html.WithUnsafe()` so author-authored raw HTML in trusted local markdown passes through (docs are the user's own repo files, per DESIGN §13's no-mutation / local-first stance). Revisit if untrusted markdown is ever rendered.
