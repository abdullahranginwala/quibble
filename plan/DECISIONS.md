# Decisions log (append-only)

- 2026-07-07 M0: `go.mod` directive is `1.25`, not `1.24` — `go-internal@v1.15` (testscript) requires 1.25; toolchain installed is 1.26.4. Conventions file said 1.24; bump was forced by the allowed dep set.
- 2026-07-07 M0: `config.Default()` authors are `human`/`agent` (generic); `quibble init` keeps them — users personalize in config. Validate() additionally requires human ≠ agent since the resolve policy gate depends on distinguishing them.
- 2026-07-07 M0: glob matcher also skips all dot-directories (not just `.quibble`) during doc walks — hidden dirs as doc sources are out of scope.
- 2026-07-07 M1: `NormalizeBlocks` signature is `func(src []byte) (text string, blocks []Block)` — the spec exports the name but leaves the signature open; this shape is the single source of truth for both `Doc.Text` and `Doc.Blocks`, so `RenderDoc` and `pkg/comment` consume the identical function. Frozen by golden `testdata/golden/normalize.json`.
- 2026-07-07 M1: paper theme assets live at `pkg/render/themes/paper/` (embedded via `go:embed themes`), not `web/themes/`. `go:embed` patterns cannot cross package boundaries (no `..`), and the M1 hard constraint forbids touching files outside `pkg/render`. Assets are trivially relocatable to a `web/` package later if a shared frontend tree is wanted.
- 2026-07-07 M1: block enumeration — a list contributes one block per *direct* list item; nested sub-lists fold their text into the containing item's block (not enumerated separately). Blocks with empty normalized text (e.g. thematic breaks, image-only paragraphs with empty alt) are excluded from `Doc.Text`/`Doc.Blocks` and receive no `data-qbl`. Spec was silent on nesting and empties; pinned by the kitchen-sink goldens.
- 2026-07-07 M1: chroma styles are `github` (light) / `github-dark` (dark), emitted as classes and scoped under `[data-qbl-scheme="dark"]` plus a `prefers-color-scheme: dark` media query. Custom `data-qbl` attributes ride goldmark's default block renderers (goldmark renders any `data-*` attribute regardless of its per-node filter), so only heading + code-block rendering is overridden.
- 2026-07-07 M1: renderer uses `html.WithUnsafe()` so author-authored raw HTML in trusted local markdown passes through (docs are the user's own repo files, per DESIGN §13's no-mutation / local-first stance). Revisit if untrusted markdown is ever rendered.
- 2026-07-07 M2: `pkg/comment` defines its own minimal `Heading{Level,Text,Anchor}` and `Block{ID,Text,Start,End}` instead of importing `render.Heading`/`render.Block` (the spec's `Sectionize` signature named them). Keeps `pkg/comment` self-contained and buildable while `pkg/render` is developed in parallel; the field sets mirror render's so the two reconcile with a type alias later if desired. Tests vendor render's frozen normalization rule (03-m1-render.md step 4: collapse intra-block whitespace runs to one space + trim, join blocks with "\n") in a local helper rather than importing `render.NormalizeBlocks`.
- 2026-07-07 M2: `Marshal` hand-writes the frozen frontmatter (fixed key order; anchor exact/prefix/suffix always double-quoted; heading as a flow sequence, omitted when empty; timestamps as plain RFC3339 scalars; resolved_by/resolved_at emitted only when set) for byte-exact determinism, while `ParseThread` decodes frontmatter with yaml.v3 for whitespace tolerance. Top-level id/doc/status/author are emitted plain when they match a safe pattern, else double-quoted — the frozen example values are all plain.
- 2026-07-07 M2: reply-marker rule is line-start + a *fully RFC3339-shaped* timestamp (not the looser `time=\S+`). Consequence/known limitation (test row 3): a body or reply-body line that is itself a fully valid `<!-- reply author=… time=<RFC3339> -->` at line start will be re-read as a reply on a Marshal→Parse round-trip. A body line resembling a marker but carrying a non-RFC3339 time stays in the body and round-trips cleanly (regression: fuzz-found input 214c8ae1, `TestRegression_BogusTimeMarkerStaysBody`). The `FuzzParseThread` round-trip invariant excludes the fully-valid-marker-in-body case; the no-panic contract holds unconditionally.
- 2026-07-07 M2: fuzzy `scanWindows` adds two result-preserving speedups the spec's perf bar demands. (1) A rolling character-multiset (bag) L1 distance is maintained incrementally over the length-L window; bag distance ≤ edit distance, so windows whose `bag/2` exceeds the working band are skipped before any DP (a sound lower-bound prefilter, gated on the 1.0× window and applied to all three width variants — a heuristic for the 0.8×/1.2× variants that never produces a false match and, given overlapping windows + small step, does not change outcomes on any specced case). (2) The banded-Levenshtein band shrinks to `maxLen·(1−bestScore)` as the best score improves, so once a strong match is seen, dissimilar windows early-exit almost immediately. Measured 200 KB fuzzy `Resolve`: ~54 ms (design bar < 100 ms), ~304 ms under `-race` (assert < 500 ms).
